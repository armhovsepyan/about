1

Переменная и память

Когда вы создаёте переменную, компьютер выделяет ей место в памяти — ячейку с адресом.

int x = 42;
// x — имя ячейки
// 42 — значение в ней
// &x == 0x7ffd1234 — адрес ячейки
x
42
0x7ffd1234
имя переменной
значение
адрес в памяти
Итог: переменная x — это имя ячейки памяти. В ней хранится значение 42, и у неё есть адрес.
2

* — Звёздочка (Указатель)

* в объявлении означает «это указатель» — переменная, которая хранит не значение, а адрес другой переменной.

int  x = 42;
int* p = &x;    // p хранит АДРЕС переменной x
                 // &x — «взять адрес» x
x
42
0x7ffd1234
←
p
0x7ffd1234
0x7ffd5678
p — отдельная переменная, у которой свой адрес (0x7ffd5678).
Но внутри неё лежит адрес x — то есть p «смотрит» на x.
Итог: int* p — переменная p хранит адрес. &x — взять адрес переменной x. Теперь p знает, где живёт x.
3

* в выражении — Разыменование

Когда * стоит перед указателем в выражении — это значит «иди по адресу и возьми значение».

int  x  = 42;
int* p  = &x;

int  v  = *p;    // *p == 42  (прочитать значение)
*p = 100;        // x теперь == 100 (записать через указатель)
x
100
0x7ffd1234
←
p
0x7ffd1234
0x7ffd5678
*p = 100 — мы не меняем сам p (он всё ещё смотрит на x). Мы меняем то, что по адресу, куда он смотрит.
Итог: *p в выражении — «зайти в ячейку, на которую указывает p». *p = 100 меняет x через указатель.
4

& — Ссылка

Ссылка — это псевдоним (второе имя) для той же переменной. Это НЕ новая ячейка памяти.

int  x = 42;
int& r = x;       // r — псевдоним x, тот же адрес!

r = 99;           // x тоже стал 99
// &r == &x  — одинаковый адрес
x / r — одна ячейка
99
0x7ffd1234
Ссылка всегда инициализируется и не может быть переназначена на другой объект — это её отличие от указателя.
Итог: int& r = x — r и x это одна и та же ячейка памяти. Изменил r — изменился x, и наоборот.
5

** — Указатель на указатель

** — это указатель, который хранит адрес другого указателя (двойная косвенность).

int   x  = 42;
int*  p  = &x;    // p  → x
int** pp = &p;    // pp → p → x

**pp == 42        // два перехода: pp→p, затем p→x
*pp  == p         // один переход: pp→p
x
42
0x1234
←
p
0x1234
0x5678
←
pp
0x5678
0x9abc
Применяется в C-стиле для динамических массивов строк (char**), двумерных массивов, и функций, которым нужно изменить указатель снаружи.
Итог: pp → хранит адрес p. p → хранит адрес x. **pp — это и есть 42. Цепочка из двух переходов.
6

*& — Ссылка на указатель

int*& rp = p — это ссылка на указатель. Позволяет изменить сам указатель p внутри функции.

void changePtr(int*& rp) {
    rp = nullptr;   // меняем сам указатель снаружи!
}

int  x = 42;
int* p = &x;
changePtr(p);
// теперь p == nullptr
x
42
0x1234
p / rp
⇄
один и тот же
p (и rp — псевдоним)
0x1234
0x5678
Читайте справа налево: int*& rp → «rp — это ссылка (&) на указатель (*) на int»
int&* — невалидный синтаксис! Нельзя сделать указатель на ссылку. Ссылка не самостоятельный объект — у неё нет своего адреса для указателя.
Итог: int*& rp = p — rp это псевдоним самой переменной p. Изменяя rp, мы меняем куда смотрит p.
f()

Поведение в параметрах функций

Случай 1
void f(int a)

Передача по значению. Получает копию. Оригинал не меняется.

Случай 2
void f(int* a)

Передача указателя. Может менять *a, но не сам указатель снаружи.

Случай 3
void f(int& a)

Передача по ссылке. Псевдоним оригинала. Удобнее и безопаснее указателя.

Случай 4
void f(int*& a)

Ссылка на указатель. Может изменить сам указатель снаружи — переключить на другой объект.

// Демо всех четырёх случаев:
int  x = 10;
int  y = 20;
int* p = &x;

void byVal(int a)    { a = 99; }       // x не изменился
void byPtr(int* a)   { *a = 99; }      // x == 99 теперь
void byRef(int& a)   { a = 99; }       // x == 99 теперь
void byPRef(int*& a) { a = &y; }      // p теперь смотрит на y!
Синтаксис Меняет *a? Меняет a (указатель)? Нужен & при вызове?
int a не применимо ✗ ✗
int* a ✓ ✗ нет (передаётся указатель)
int& a ✓ нет (ссылка) ✗
int*& a ✓ ✓ нет (передаётся указатель)
C

const — Защита данных

const защищает то, что стоит справа от него. Читайте объявление справа налево — всё встаёт на место.

Простые случаи

const int  x = 42;   // x нельзя изменить
int const  y = 42;   // то же самое

const + указатель — самая путаная часть

// Читаем справа налево:

const int* p1;
// p1 — указатель на const int
// нельзя менять *p1, но можно p1 = &другой_переменной

int* const p2 = &x;
// p2 — const указатель на int
// нельзя менять p2 (куда смотрит), но *p2 = 5 — можно

const int* const p3 = &x;
// p3 — const указатель на const int
// нельзя ни *p3 менять, ни сам p3
const int* p
❌ *p = x
✅ p = &y
int* const p
✅ *p = x
❌ p = &y
const int* const p
❌ *p = x
❌ p = &y

const в параметрах функций — самое важное на практике

void print(const int& val)  // читаем: val — ссылка на const int
{
    // val = 5;  — ошибка компиляции!
    // Гарантируем вызывающему: мы не изменим его данные.
}
// Это самый частый паттерн C++ для больших объектов:
// передаём без копирования + гарантируем неизменность.

void process(const int* p)  // указатель на const int
{
    // *p = 5;  — ошибка!
    // p = &x;  — можно (сам указатель не защищён)
}
Объявление Читается как *p менять? p менять?
int* pуказатель на int✓✓
const int* pуказатель на const int✗✓
int* const pconst-указатель на int✓✗
const int* const pconst-указатель на const int✗✗
const int& rссылка на const int✗н/п
int& const rбессмысленно — ошибка!––
Три главных правила:
1. const защищает то, что стоит справа от него.
2. const int& — самый частый паттерн C++: передать без копирования, запретить изменение.
3. int& const — бессмысленно и выдаст ошибку. Ссылка и так не переназначается.

Итоговая шпаргалка

Синтаксис Название Что хранит Типичное применение
int x Переменная Значение Обычные данные
int* p Указатель Адрес переменной Динамическая память, массивы
&x Взять адрес — Получить адрес переменной
*p Разыменование — Прочитать/записать по адресу
int& r Ссылка Псевдоним Параметры функций, range-for

Главная подсказка для чтения: читайте объявление справа налево.

Пример: int*& rp — читаем:
«rp — это ссылка (&) на указатель (*) на int»

Пример: const int* const p — читаем:
«p — это const-указатель (*) на const-int»
Быстрая справка — формула памяти
int* → указатель
int& → ссылка
int** → указатель на указатель
int*& → ссылка на указатель
const int* → указатель на const
int* const → const указатель
