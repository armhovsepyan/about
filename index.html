Переменная и память

Когда вы создаёте переменную, компьютер выделяет ей место в памяти — ячейку с адресом.

int x = 42;
// x — имя ячейки
// 42 — значение в ней
// &x == 0x7ffd1234 — адрес ячейки
x
42
0x7ffd1234
имя переменной
значение
адрес в памяти
Итог: переменная x — это имя ячейки памяти. В ней хранится значение 42, и у неё есть адрес.
2

* — Звёздочка (Указатель)

* в объявлении означает «это указатель» — переменная, которая хранит не значение, а адрес другой переменной.

int  x = 42;
int* p = &x;    // p хранит АДРЕС переменной x
                 // &x — «взять адрес» x
x
42
0x7ffd1234
←
p
0x7ffd1234
0x7ffd5678
p — отдельная переменная, у которой свой адрес (0x7ffd5678).
Но внутри неё лежит адрес x — то есть p «смотрит» на x.
Итог: int* p — переменная p хранит адрес. &x — взять адрес переменной x. Теперь p знает, где живёт x.
3

* в выражении — Разыменование

Когда * стоит перед указателем в выражении — это значит «иди по адресу и возьми значение».

int  x  = 42;
int* p  = &x;

int  v  = *p;    // *p == 42  (прочитать значение)
*p = 100;        // x теперь == 100 (записать через указатель)
x
100
0x7ffd1234
←
p
0x7ffd1234
0x7ffd5678
*p = 100 — мы не меняем сам p (он всё ещё смотрит на x). Мы меняем то, что по адресу, куда он смотрит.
Итог: *p в выражении — «зайти в ячейку, на которую указывает p». *p = 100 меняет x через указатель.
4

& — Ссылка

Ссылка — это псевдоним (второе имя) для той же переменной. Это НЕ новая ячейка памяти.

int  x = 42;
int& r = x;       // r — псевдоним x, тот же адрес!

r = 99;           // x тоже стал 99
// &r == &x  — одинаковый адрес
x / r — одна ячейка
99
0x7ffd1234
Ссылка всегда инициализируется и не может быть переназначена на другой объект — это её отличие от указателя.
Итог: int& r = x — r и x это одна и та же ячейка памяти. Изменил r — изменился x, и наоборот.
5

** — Указатель на указатель

** — это указатель, который хранит адрес другого указателя (двойная косвенность).

int   x  = 42;
int*  p  = &x;    // p  → x
int** pp = &p;    // pp → p → x

**pp == 42        // два перехода: pp→p, затем p→x
*pp  == p         // один переход: pp→p
x
42
0x1234
←
p
0x1234
0x5678
←
pp
0x5678
0x9abc
Применяется в C-стиле для динамических массивов строк (char**), двумерных массивов, и функций, которым нужно изменить указатель снаружи.
Итог: pp → хранит адрес p. p → хранит адрес x. **pp — это и есть 42. Цепочка из двух переходов.
6

*& — Ссылка на указатель

int*& rp = p — это ссылка на указатель. Позволяет изменить сам указатель p внутри функции.

void changePtr(int*& rp) {
    rp = nullptr;   // меняем сам указатель снаружи!
}

int  x = 42;
int* p = &x;
changePtr(p);
// теперь p == nullptr
x
42
0x1234
p / rp
⇄
один и тот же
p (и rp — псевдоним)
0x1234
0x5678
Читайте справа налево: int*& rp → «rp — это ссылка (&) на указатель (*) на int»
int&* — невалидный синтаксис! Нельзя сделать указатель на ссылку. Ссылка не самостоятельный объект — у неё нет своего адреса для указателя.
Итог: int*& rp = p — rp это псевдоним самой переменной p. Изменяя rp, мы меняем куда смотрит p.
f()

Поведение в параметрах функций

Случай 1
void f(int a)

Передача по значению. Получает копию. Оригинал не меняется.

Случай 2
void f(int* a)

Передача указателя. Может менять *a, но не сам указатель снаружи.

Случай 3
void f(int& a)

Передача по ссылке. Псевдоним оригинала. Удобнее и безопаснее указателя.

Случай 4
void f(int*& a)

Ссылка на указатель. Может изменить сам указатель снаружи — переключить на другой объект.

// Демо всех четырёх случаев:
int  x = 10;
int  y = 20;
int* p = &x;

void byVal(int a)    { a = 99; }       // x не изменился
void byPtr(int* a)   { *a = 99; }      // x == 99 теперь
void byRef(int& a)   { a = 99; }       // x == 99 теперь
void byPRef(int*& a) { a = &y; }      // p теперь смотрит на y!
Синтаксис Меняет *a? Меняет a (указатель)? Нужен & при вызове?
int a не применимо ✗ ✗
int* a ✓ ✗ нет (передаётся указатель)
int& a ✓ нет (ссылка) ✗
int*& a ✓ ✓ нет (передаётся указатель)
C

const — Защита данных

const защищает то, что стоит справа от него. Читайте объявление справа налево — всё встаёт на место.

Простые случаи

const int  x = 42;   // x нельзя изменить
int const  y = 42;   // то же самое

const + указатель — самая путаная часть

// Читаем справа налево:

const int* p1;
// p1 — указатель на const int
// нельзя менять *p1, но можно p1 = &другой_переменной

int* const p2 = &x;
// p2 — const указатель на int
// нельзя менять p2 (куда смотрит), но *p2 = 5 — можно

const int* const p3 = &x;
// p3 — const указатель на const int
// нельзя ни *p3 менять, ни сам p3
const int* p
❌ *p = x
✅ p = &y
int* const p
✅ *p = x
❌ p = &y
const int* const p
❌ *p = x
❌ p = &y

const в параметрах функций — самое важное на практике

void print(const int& val)  // читаем: val — ссылка на const int
{
    // val = 5;  — ошибка компиляции!
    // Гарантируем вызывающему: мы не изменим его данные.
}
// Это самый частый паттерн C++ для больших объектов:
// передаём без копирования + гарантируем неизменность.

void process(const int* p)  // указатель на const int
{
    // *p = 5;  — ошибка!
    // p = &x;  — можно (сам указатель не защищён)
}
Объявление Читается как *p менять? p менять?
int* pуказатель на int✓✓
const int* pуказатель на const int✗✓
int* const pconst-указатель на int✓✗
const int* const pconst-указатель на const int✗✗
const int& rссылка на const int✗н/п
int& const rбессмысленно — ошибка!––
Три главных правила:
1. const защищает то, что стоит справа от него.
2. const int& — самый частый паттерн C++: передать без копирования, запретить изменение.
3. int& const — бессмысленно и выдаст ошибку. Ссылка и так не переназначается.

C++11

C++11 — ISO/IEC 14882:2011  "Modern C++ begins"

Революционный стандарт — Страуструп назвал его «другим языком». Move-семантика устранила лишние копирования и перевернула дизайн типов; lambda-функции похоронили verbose функторы; smart pointers взяли управление памятью на себя. Многопоточность впервые стала частью стандарта — единая модель памяти, std::thread, std::mutex, std::atomic. Именно C++11 превратил C++ в современный язык и заложил фундамент, на котором строится всё последующее.

Core Language

auto — вывод типа / decltype — тип выражения

auto x  = 42;           // int
auto d  = 3.14;         // double
auto it = v.begin();   // std::vector<int>::iterator — без длинного типа

// decltype — тип выражения без его вычисления
int a = 5;
decltype(a)       b = 10;  // int
decltype(a + 0.5) c;       // double

// trailing return type с decltype:
template<class A, class B>
auto add(A a, B b) -> decltype(a + b) { return a + b; }

Range-based for + Uniform initialization + Initializer lists

// {} — единый синтаксис инициализации для всего
int              arr[] = {1, 2, 3};
std::vector<int>  v   = {4, 5, 6};        // std::initializer_list
std::map<int,int> m   = {{1,10},{2,20}};

// Range-based for
for (const auto& x : v)
    std::cout << x;  // 4 5 6

Lambda expressions

// [захват](параметры) -> тип { тело }
auto add   = [](int a, int b) { return a + b; };

int base = 10;
auto byVal = [base](int x)  { return base + x; };  // захват по значению
auto byRef = [&base](int x) { base += x; };         // захват по ссылке
auto allv  = [=](int x)    { return base + x; };   // всё по значению
auto allr  = [&](int x)    { base += x; };          // всё по ссылке

std::sort(v.begin(), v.end(), [](int a, int b){ return a > b; });

Move semantics && rvalue references

// T&& — привязывается к временному/перемещаемому объекту
std::string s1 = "hello";
std::string s2 = std::move(s1);  // s1 пуст — данные ПЕРЕМЕЩЕНЫ, не скопированы

// Move-конструктор
struct Buf {
    Buf(Buf&& o) noexcept : data(o.data) { o.data = nullptr; }
    int* data;
};

// std::forward — perfect forwarding, сохраняет lvalue/rvalue-категорию
template<typename T>
void wrapper(T&& val) { process(std::forward<T>(val)); }

constexpr — вычисления в compile-time

constexpr int square(int x) { return x * x; }

constexpr int SIZE = square(8);  // 64 — вычислено компилятором
int arr[SIZE];                    // OK: SIZE известен в compile-time

constexpr double PI = 3.14159265358979;

Variadic templates

// ... — пакет параметров произвольной длины
// Рекурсивный стиль (C++11):
void print() {}
template<typename T, typename... Rest>
void print(T first, Rest... rest) {
    std::cout << first; print(rest...);
}
print(1, " x ", 3.14);  // 1 x 3.14

// sizeof...(pack) — количество элементов в пакете
template<typename... T> constexpr auto count = sizeof...(T);

nullptr, override, final, enum class, noexcept, static_assert

int* p = nullptr;           // типобезопасный нулевой указатель (не NULL/0)

struct Base  { virtual void foo(int) {} };
struct Child : Base { void foo(int) override {} };  // ошибка если сигнатура не совпадает
struct Leaf  final : Child {};                       // запрет дальнейшего наследования

enum class Color { Red, Green, Blue };
Color c = Color::Red;         // нет неявного int; квалификатор обязателен

void safe() noexcept {}      // гарантия: не бросает исключений
static_assert(sizeof(int) == 4, "int must be 4 bytes"); // compile-time проверка

Delegating constructors, explicit conversion, raw strings, UDL

// Делегирующий конструктор — вызывает другой конструктор этого же класса
struct Foo {
    Foo() : Foo(0) {}      // делегирует в Foo(int)
    Foo(int x) : val(x) {}
    int val;
};

// explicit для операторов конверсии — нет случайных преобразований
struct Safe { explicit operator bool() const { return valid; } bool valid; };

// Raw string literals — без escape-последовательностей
std::string path = R"(C:\Users\name\file.txt)";
std::string re   = R"(\d+\.\d+)";

// User-defined literals — собственные суффиксы
constexpr long double operator""_km(long double d) { return d * 1000; }
auto dist = 1.5_km;  // 1500.0
alignas / alignof — управление выравниванием типов и переменных
Inline namespaces — вложенные версии API с ADL-прозрачностью
[[attr]] — расширяемый синтаксис атрибутов компилятора
char16_t / char32_t — Unicode типы; литералы u"...", U"..."
Standard Library

Smart pointers: unique_ptr, shared_ptr, weak_ptr

// unique_ptr — единственный владелец, RAII, без накладных расходов
std::unique_ptr<int> u(new int(42));  // delete не нужен

// shared_ptr — счётчик ссылок, несколько совладельцев
auto s1 = std::make_shared<std::string>("hello");
auto s2 = s1;   // count == 2
s1.reset();     // count == 1
s2.reset();     // count == 0; объект удалён

// weak_ptr — наблюдатель без владения (разрывает циклические ссылки)
std::weak_ptr<std::string> w = s1;
if (auto lock = w.lock())  // получить shared_ptr если объект ещё жив
    std::cout << *lock;

std::thread, std::mutex, std::condition_variable

std::mutex mtx;  int shared = 0;

std::thread t1([&]() {
    std::lock_guard<std::mutex> lock(mtx);  // RAII-блокировка
    shared++;
});
std::thread t2([&]() {
    std::unique_lock<std::mutex> lock(mtx); // гибкий вариант (можно unlock вручную)
    shared++;
});
t1.join(); t2.join();

// condition_variable — ожидание условия с автоматическим unlock
std::condition_variable cv;
cv.wait(lock, []{ return ready; });  // атомарно: unlock + sleep; wake-up + relock

std::async / std::future / std::promise

// async — запуск задачи в пуле потоков
std::future<int> f = std::async(std::launch::async, []{ return 42; });
int result = f.get();  // блокирует до готовности

// promise — ручная передача значения между потоками
std::promise<std::string> p;
std::future<std::string>  fut = p.get_future();
std::thread([&p]{ p.set_value("done"); }).detach();
std::cout << fut.get();  // "done"

std::array, std::unordered_map/set, std::tuple

// std::array — C-массив с методами STL, без heap-аллокации
std::array<int, 3> arr = {1, 2, 3};
arr.size(); arr.fill(0); arr.data();

// std::unordered_map — хэш-таблица, O(1) среднее время
std::unordered_map<std::string, int> m;
m["a"] = 1;

// std::tuple — гетерогенный набор значений
auto t = std::make_tuple(1, "hi", 3.14);

int a; std::string b;
std::tie(a, b, std::ignore) = t;
std::function — обёртка над любым вызываемым (функция, лямбда, метод)
std::bind — привязка аргументов; частичное применение функции
std::regex — регулярные выражения (regex_match, regex_search)
std::chrono — duration, time_point, steady_clock, system_clock
std::begin / std::end — ADL-совместимый доступ к диапазонам
std::move / std::forward — утилиты move-семантики
<type_traits> — is_integral_v, enable_if, conditional, decay и др.
std::atomic<T> — атомарные операции без mutex; поддержка memory_order
std::unordered_set — хэш-множество O(1)
C++11: auto/decltype, lambda, move&&, constexpr, variadic templates, nullptr, override/final, enum class, noexcept — фундамент всего современного C++.
C++14

C++14 — ISO/IEC 14882:2014  "Polishing C++11"

Небольшой, но важный стандарт — «полировка» C++11, устраняющая практические ограничения. Generic lambdas с auto-параметрами избавили от необходимости писать шаблонные функторы вручную; relaxed constexpr позволил использовать циклы и условия в compile-time функциях. Variable templates упростили параметризованные константы вроде pi<float>. Добавлен недостающий std::make_unique, UDL для <chrono> (1h, 500ms) и <string> ("hello"s).

Core Language

Generic lambdas + Lambda init-captures

// C++14: auto-параметры — лямбда становится шаблонной функцией
auto add = [](auto a, auto b) { return a + b; };
add(1, 2);                 // int
add(1.5, 2.5);            // double
add(std::string{"a"}, std::string{"b"}); // string

// Lambda init-capture — создание/инициализация переменной прямо в захвате
auto ptr = std::make_unique<int>(42);
auto f   = [p = std::move(ptr)]() { return *p; };  // перемещение в захват

Return type deduction + Relaxed constexpr

// Компилятор выводит тип возврата из return-выражений
auto mul(int a, int b) { return a * b; }            // → int
auto wrap(int x)       { return std::pair{x, x*2}; }  // → pair<int,int>

// Relaxed constexpr: C++11 запрещал циклы и условия — C++14 снял ограничение
constexpr int factorial(int n) {
    int r = 1;
    for (int i = 2; i <= n; ++i) r *= i;
    return r;
}
constexpr int f6 = factorial(6);  // 720 — в compile-time

Variable templates

// Шаблонные переменные — параметризованные константы
template<typename T>
constexpr T pi = T(3.14159265358979);

auto a = pi<float>;    // 3.14159f
auto b = pi<double>;  // 3.14159265358979
auto c = pi<int>;     // 3

Binary literals + Digit separators

// Двоичные литералы с префиксом 0b
int flags = 0b1010'1100;     // = 172

// Разделитель разрядов ' — только для читаемости, на значение не влияет
long   million = 1'000'000;
double pi      = 3.141'592'653;
int    mask    = 0xFF'FF'FF'FF;

Standard Library

std::make_unique + std::integer_sequence + std::exchange

// make_unique — безопасно и симметрично с make_shared (не было в C++11!)
auto p = std::make_unique<std::string>("hello");

// integer_sequence — список индексов в compile-time (для tuple и pack expansion)
template<typename Tuple, std::size_t... I>
void printTuple(const Tuple& t, std::index_sequence<I...>) {
    ((std::cout << std::get<I>(t) << ' '), ...);
}

// exchange — атомарная замена: вернуть старое, записать новое
int old = std::exchange(x, newVal);  // old = x; x = newVal;
Heterogeneous lookup в std::map/set: поиск без создания объекта ключа (find("key") не создаёт std::string)
std::shared_timed_mutex — read-write мьютекс (shared для чтения, exclusive для записи)
UDL для <chrono>: 1h, 30min, 500ms, 1us, 1ns
UDL для <string>: суффикс s создаёт std::string: "hello"s
C++14: generic lambdas, init-captures, relaxed constexpr, variable templates, make_unique, binary literals — небольшой, но важный шаг.
C++17

C++17 — ISO/IEC 14882:2017  "Big quality-of-life update"

Крупное обновление с акцентом на эргономику кода. Structured bindings устранили синтаксический шум при работе с парами и картами; if constexpr заменил громоздкий SFINAE в большинстве шаблонных задач; fold expressions сделали variadic код читаемым без рекурсии. В библиотеке наконец появились std::optional, std::variant, std::string_view и std::filesystem — вещи, которых не хватало годами, — а параллельные алгоритмы (std::execution::par) добавили многопоточность одной строкой.

    else if constexpr (std::is_floating_point_v<T>)
        std::cout << "float: " << val;
    else
        std::cout << "other";

Fold expressions

C++20

C++20 — ISO/IEC 14882:2020  "Biggest since C++11"

Самый крупный стандарт со времён C++11 — четыре кита. Concepts сделали шаблоны читаемыми и дают понятные ошибки компиляции вместо стены текста. Ranges принесли ленивые конвейеры над данными через оператор |. Coroutines добавили co_await / co_yield для async-кода и генераторов без колбэков. Modules заменили #include изолированной системой с гарантированным ODR. В довесок: spaceship <=> автоматически генерирует все операторы сравнения, std::format даёт type-safe форматирование, consteval гарантирует compile-time выполнение.

Core Language

Concepts — ограничения шаблонов

// До C++20: ошибки шаблонов — стена нечитаемого текста
template<typename T>
concept Numeric = std::is_arithmetic_v<T>;

template<Numeric T>            // T обязан быть числовым
T square(T x) { return x * x; }

square(5);       // OK
square("hi");   // ошибка — читаемое сообщение: "string не удовлетворяет Numeric"

// requires-expression — произвольные ограничения на операции
template<typename T>
concept Addable = requires(T a, T b) { a + b; };

// Краткий синтаксис (abbreviated template):
auto square(Numeric auto x) { return x * x; }

Modules — замена #include

// mymath.ixx — объявление модуля
export module mymath;

export int square(int x) { return x * x; }
export double pi = 3.14159;

int helper() { return 0; }  // не экспортируется — скрыт от клиентов

// main.cpp
import mymath;
import std;  // весь стандарт как один модуль (C++23 практика)

int main() { std::println("{}", square(5)); }

Coroutines — co_await, co_yield, co_return

// Функция-coroutine: может приостановиться и возобновиться
// (Требует coroutine-фреймворк; ниже показан std::generator из C++23)

// co_yield — генератор значений
std::generator<int> range(int from, int to) {
    for (int i = from; i < to; ++i)
        co_yield i;         // приостановиться, вернуть i
}
for (int x : range(0, 5)) std::cout << x;  // 0 1 2 3 4

// co_await — асинхронное ожидание (в async-фреймворке)
Task<int> fetch() {
    auto data = co_await httpGet("/api");
    co_return process(data);
}

C++23

C++23 — ISO/IEC 14882:2023  "Current standard"

Текущий стандарт, продолжающий путь C++20. std::expected<T,E> привнёс функциональный стиль обработки ошибок без исключений — явный тип вместо errno или кодов возврата. Deducing this (explicit object parameter) устранил дублирование const/non-const перегрузок и открыл рекурсивные lambdas. std::print/println объединили std::format и вывод в один вызов. Ranges получили zip, enumerate, chunk, slide, cartesian_product и ranges::to для материализации ленивых view в контейнеры.

Core Language

Deducing this — explicit object parameter

// this передаётся как явный параметр — устраняет дублирование const/non-const версий
struct Widget {
    template<typename Self>
    auto& value(this Self&& self) { return self.m_value; }
    int m_value = 0;
};

// Рекурсивные лямбды — главный use case:
auto factorial = [](this auto self, int n) -> int {
    return n <= 1 ? 1 : n * self(n - 1);
};
factorial(5);  // 120

if consteval + auto в параметрах функций

// if consteval — ветвление по факту compile-time/runtime
constexpr int get(int x) {
    if consteval { return x * 2; }  // compile-time путь
    else          { return x + 1; }  // runtime путь
}

// auto в обычных параметрах функций (не только lambdas/templates)
void print(auto val) { std::cout << val; }
// Эквивалентно: template<typename T> void print(T val)
